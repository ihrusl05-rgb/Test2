**Что работает сейчас**:

-Регистрация и вход пользователей
-Каталог товаров с категориями
-Поиск по названию, описанию и ID
-Пагинация (когда много товаров)
-Фильтрация по категориям
-Модальные окна с деталями товара (пока без API, просто рендер)
-Админка для управления
-Защита страниц — незалогиненных швыряет на вход

**Структура проекта**
app1/
├── app/              # настройки Django
├── users/            # пользователи, авторизация
├── partners/         # каталог, категории, товары
├── templates/        # общие шаблоны
├── static/           # css, js, картинки
├── media/            # загруженные файлы
└── fixtures/         # тестовые данные. отсюда грузим категории и товары через loaddata


**Быстрый старт**

# Клонируем, ставим зависимости.

В settings.py меняем базу на SQLite для проверки:

DATABASES = {
    "default": {
        "ENGINE": "django.db.backends.sqlite3",
        "NAME": "mydatabase",
    }
}
(документация по настройкам: https://docs.djangoproject.com/en/6.0/ref/settings/#databases)

# Дальше стандартно:

```console
python manage.py migrate
python manage.py createsuperuser
python manage.py runserver
```

# Тестовые данные (необязательно)


```console
python manage.py loaddata fixtures/partners/categories.json
python manage.py loaddata fixtures/partners/products.json
```
